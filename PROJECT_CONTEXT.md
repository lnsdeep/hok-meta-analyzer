# Project Context & AI Instructions

## 📖 Project Overview
The **Honor of Kings Hero Meta Database** is a single-page web dashboard designed to track Hero statistics (Win Rate, Pick Rate, Ban Rate) and Tiers for the mobile game Honor of Kings (HoK). 
Its primary goal is bridging the gap for global/English-speaking players by marrying the official Chinese real-time metrics with English hero names and epithets.

## 🛠 Tech Stack
- **Frontend**: Vanilla HTML5, CSS3, ES6 JavaScript (No frameworks like React or Vue).
- **Backend/Scraper**: Python 3, `playwright` (async), `asyncio`.
- **Hosting**: Designed for GitHub Pages (with GitHub Actions automating the scraping).

## 📂 File Structure & Responsibilities
- **`index.html`**: The monolithic frontend file. 
  - Contains CSS for styling (dark mode, premium aesthetic, CSS grid for mobile cards).
  - Contains Vanilla JS for fetching JSON, rendering the DOM table/cards, search filtering, dropdown filtering, and URL parameter logic (`history.pushState`).
- **`scraper_unified.py`**: The core data ingestion engine.
  - Concurrently scrapes Tencent's official *Camp HOK* for real-time tier/popularity and win/pick/ban metrics.
  - Uses a local CSV mapping (`hero_mappings.csv`) to resolve official English character names/epithets.
  - Uses `asyncio.Semaphore(5)` to prevent overloading websites and avoids getting blocked.
- **`heroes.json` & `heroes.csv`**: The parsed database output generated by the scraper. The UI strictly reads from `heroes.json`.
- **`metadata.json`**: Generated by the scraper to track the UTC time of the last update. Parsed by the UI to show localized update times.
- **`.github/` (Workflows)**: Automation scripts (GitHub Actions) to run the scraper on a CRON schedule and commit the updated JSON/CSV files.
- **`README.md`**: Public-facing repository documentation.

## 🧠 Key Logic & Quirks
1. **Frontend Mobile Design**: Under a 768px width viewport, `index.html` uses heavy media queries to dismantle the HTML `<table>` into responsive, stacked CSS Grid cards. Any modifications to the table HTML must reflect in these CSS overrides.
2. **Frontend Sorting Logic**: Sorting by "Tier" includes a multi-level tie-breaker (First sorts by Tier [S, A, B, C, D], then resolves ties using Win Rate). 
3. **Scraper Resiliency**: `scraper_unified.py` relies on `safe_goto` which implements exponential backoff and retries. This is crucial for dealing with HTTP 429 (Too Many Requests) or timeouts. 
4. **Data Syncing**: The scraper overrides `heroes.json` completely on each run.

## 🤖 Instructions for LLMs / AI Agents
When assisting with this project, please adhere to the following rules:
1. **No Frontend Frameworks**: Do not suggest or implement React, Vue, Tailwind, or jQuery. Keep it 100% Vanilla HTML/CSS/JS.
2. **Mobile-First CSS**: When adding new columns or buttons to the frontend table, **always** consider how it will look in the mobile `@media (max-width: 768px)` break point. You must manually map the CSS Grid to fit new data.
3. **Maintain Premium Aesthetics**: Keep UI changes aligned with the existing dark-mode palette (`var(--bg-color)`, `var(--card-bg)`, `var(--accent)`, etc.). Use rich layouts, border-radius, and smooth micro-animations on interactive elements.
4. **Scraper Changes**: If modifying the bot, ensure that you respect the `playwright` async patterns. Do not refactor `async` into synchronous requests unless absolutely necessary. Be mindful of bot detection / rate-limits.
5. **English/Pinyin Dual Names**: Always keep the context that players search using English, but the underlying base data often uses Pinyin or Chinese names as the ID or initial source.
